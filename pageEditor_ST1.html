Template & Global CSS

HTML Structure

I built a reusable structure using semantic elements: <header> holds the brand identity and the <nav>, <main> wraps each page's bespoke content, and <footer> carries per-student Page Editor, Validation and compliance links. Semantic tags improve both accessibility and validation.

CSS Techniques

The global stylesheet uses CSS custom properties (:root variables) for a shared palette, an 8px spacing scale, and a Fraunces + Inter type pairing. Flexbox powers the sticky translucent header; CSS Grid drives the card grids and editorial split rows. Hover states use transition, and an .active class marks the current nav item.

Rationale

Variables let a colour or spacing change cascade everywhere without editing each page, supporting visual consistency and the site-wide "guardian's journey" story. A prefers-reduced-motion block disables animation for users who need it.

Home Page

HTML Structure

The <main> holds a hero <section>, an animated stat row, three mission <article> cards, an interactive gallery linking to all four content pages, and a call-to-action band into the next chapter.

CSS & JS Techniques

Page-specific embedded CSS (5+ rules) styles the hero overlay and gallery-tile hover animation. A small vanilla-JS IntersectionObserver drives the stat counters and scroll-reveal, all reduced-motion safe.

Gallery Page

HTML Structure

Thumbnails store their extended-view data in data-* attributes; a hidden modal <div> is populated on demand.

JavaScript & CSS Techniques

Vanilla JS event listeners handle click-to-expand, close, and click-outside behaviour, plus buttons that switch the colour scheme and font of the extended view only (with the active button highlighted). CSS handles the hover cue and modal transitions.

User Profile

HTML Structure

Instead of a form, the page uses two <section> panels: a control panel with a <progress> element, and an output panel with three hidden profile sections.

JavaScript Techniques

A profile object stores nine fields across three steps. Each step fires three prompt() windows; JS handles skips (Cancel/empty), reveals only completed sections via DOM manipulation, updates the progress bar, and lets users revisit any step to add missing details.

Sitemap

HTML & SVG Techniques

An inline <svg> with a viewBox scales responsively. Every node is an <a> wrapping a <rect> and <text>, with a <title> for its accessible name. CSS gives hover and keyboard-focus feedback, and <path> connectors routed through the gaps show hierarchy without overlapping.

Challenges & Lessons Learned

The main challenge was keeping page-specific styles from clashing with the global CSS while rolling out a whole design system. I resolved this by scoping embedded styles under page-specific class names and documenting which parts of the template must not change. I also fixed a hover-flicker bug on cards by switching from movement transforms to an absolutely-positioned overlay, and reworked the sitemap connectors so trunk lines route through gaps between nodes rather than crossing them. I learned how a shared design system keeps a multi-page, story-driven site coherent.

Accessibility Considerations

Every image has descriptive alt text, the nav uses an aria-label, thumbnails are keyboard-operable (tabindex + Enter/Space), a :focus-visible ring is provided globally, and colour contrast between text and backgrounds meets readable levels. Semantic landmarks help screen readers navigate.

Compliance with JANET Regulations

All content is original and educational, external images are royalty-free (Unsplash) and referenced, and no unlawful or offensive material is published. This respects JANET's acceptable-use expectations for responsible, lawful web publishing and academic integrity.
