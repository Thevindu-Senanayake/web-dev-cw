Feedback Page

HTML Structure

Three clearly separated blocks: a scrollable programme directory (4 cards), a ratings/reviews section, and a feedback <form> grouped with <fieldset> and <legend>. Fields include email, tel, date, datetime-local, a select and a maxlength textarea.

JavaScript Techniques

A live character counter updates as the user types, and custom JS validation checks required fields on submit, highlighting invalid inputs and showing a confirmation message on success.

Team Page

HTML & CSS Techniques

Four member <article> blocks with tabindex="0" for keyboard access. A hidden details panel slides up as an overlay on hover/focus using CSS transform and opacity transitions - implemented as an overlay (not expanding height) to avoid a hover-flicker bug at the card edge.

Content Page 3

Covers Soil & Sustainable Land with four sections, four images, anchor navigation and a CSS-only Go-to-Top button.

Challenges & Lessons Learned

Making the form validation feel helpful rather than blocking took iteration. I added inline error messages beside each field and a red highlight class. I learned how to combine native HTML attributes with JavaScript for richer feedback.

Accessibility Considerations

All inputs have associated <label> elements, the team blocks are keyboard-focusable with a visible focus outline, images carry alt text, and error messages are placed next to their fields for clarity.

Compliance with JANET Regulations

The form collects no data on a server and stores nothing, reviews are fictional and clearly illustrative, and all imagery is royalty-free and credited - consistent with responsible, lawful publishing under JANET.
